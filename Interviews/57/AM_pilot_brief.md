# AM Pilot Requirements Brief — Interview 57 (CSL)

Objective:
- Validate a small, operationally practical additive-manufacturing (AM) pilot that reduces downtime for non-critical spare parts and tooling, while preserving OEM IP and meeting classification requirements.

Scope (pilot):
- One ship (short-sea vessel) + one port-side AM partner.
- Focus on low-certification, high-value consumables and tooling (brackets, clamps, non-structural fittings, tooling jigs), not structural parts.

Success metrics:
- Time-to-delivery reduction (order → onboard use) vs current sourcing (target: 50%+ improvement).
- Verified part fit/function on first install (pass QA checklist).
- No classification or regulatory blocks during pilot (or documented mitigation).
- Clear OEM agreement for controlled file use (proof-of-concept acceptance).

Hardware & materials:
- Industrial metal AM printer at port partner (recommended: laser powder bed fusion for selected alloys).
- Controlled material storage (desiccant, sealed containers) and simple environmental checks for shipboard handling.

Software & data:
- Secure file delivery system with OEM-controlled unlock (encrypted device-interpretable files or print-unlock tokens).
- Digital traceability: build record, process parameters, certificate generation for each printed part.

Regulatory / certification:
- Engage a classification society (Lloyd's/DNV) early for acceptance criteria and digital certificate format.
- Limit pilot to non-flag-critical parts to avoid immediate class/legal risk.

Operations & skills:
- One trained technician per ship visit; remote AR-enabled support for installation/inspection.
- Simple checklists for storage, pre-print inspection, post-print QA, and fit test.

Pilot timeline (8–12 weeks):
1. Weeks 1–2: Select ship, define candidate part list, onboard stakeholders (OEM, port partner, class contact).
2. Weeks 3–5: Secure OEM file access agreements, set up secure file flow, and validate print recipes at port partner.
3. Weeks 6–8: Print parts, deliver to ship, perform installation/fitting and QA; collect metrics.
4. Weeks 9–12: Review results, iterate, prepare regulatory evidence and recommendations.

Risks & mitigations:
- OEM IP refusal — propose encrypted, device-only file format and revenue-sharing/licensing model.
- Certification delays — limit pilot to non-critical parts; engage class society upfront.
- Environmental/storage issues — define material handling SOPs and small controlled inventory limits.

Next steps:

- Draft an OEM IP protection proposal (encrypted print tokens) and a simple QA checklist for pilot parts.
